# 平台檢舉處理 SOP（內部）

> **用途**：本文件供平台與管理員內部使用，**不對外公開**。
> 它的法律意義在於：當有人檢舉違規（如違法商品、誇大療效宣稱、詐騙）時，我們能證明「**有處理、沒放任**」。真正保護平台的不是文件寫得漂亮，而是**每次都照做、而且留下紀錄**。
>
> 對外的措辭請見[使用者條款](./terms-of-service.md) 第五條 5.4——條款寫的是「平台**有權**處理、但**不保證**時效與結果」，本 SOP 寫的是「我們**實際上**怎麼做」，兩者刻意分工、不衝突。

---

## 核心原則

收到檢舉 → 依風險分級 → **高風險先下架再查** → 處置後留紀錄、通知雙方。

**寧可先下架再查證，也不要明知不管。**

---

## 處理步驟

1. **受理**：檢舉從「站內檢舉鈕」或「服務信箱」進來。一律記下：檢舉時間、檢舉人、被檢舉對象（賣家 / 商品 / 評價 / 連線）、理由。
2. **分級**：依下表決定處理優先順序與方式。
3. **先行處置**：屬高風險者，**先暫時下架 / 隱藏**該商品或內容，再進行查證（先止血，避免擴大）。
4. **查證**：檢視商品描述、檢舉內容，必要時**請賣家於期限內說明**。
5. **決定與行動**（擇一）：
   - 違規成立 → 下架商品 / 移除違規字句 / 賣家停權（依情節輕重）
   - 涉及犯罪（如詐騙）→ 保留證據，必要時配合或通報主管機關
   - 檢舉不成立 → 恢復上架，並記錄理由
6. **通知**：通知賣家處置結果與理由（賣家可申訴）；視情況回覆檢舉人「已處理」。
7. **留紀錄**：每一步的時間、決定、處理人都存檔。**這是最重要的一步。**

---

## 風險分級與處理方式

| 風險等級 | 例子 | 處理方式 |
|---------|------|---------|
| **高**（先下架再查） | 違法 / 管制品、詐騙、誇大療效或醫療宣稱、危害人身安全、仿冒 | 知悉後**儘速**先暫時下架，並於短時間內查證 |
| **一般** | 商品品質爭議、出貨慢、用語不當、口角 | 於數個工作日內審視、聯繫雙方協調 |
| **惡意 / 不實檢舉** | 競爭對手亂檢舉、查無此事 | 檢舉不成立並記錄；多次惡意者依條款停權檢舉人 |

> 時限刻意寫成「儘速 / 數個工作日」而非寫死幾小時——這是**內部目標**，寫死了反而變成做不到就打自己臉。

---

## 兩個保命重點

1. **高風險先下架**：寧可先撤、查清楚再恢復，也不要拖著不動。
2. **全程留紀錄**：誰、什麼時候、看了什麼、做了什麼決定——出事時這份紀錄就是平台的護身符。

---

## 累犯處理

同一賣家多次違規 → 升級處置：**警告 → 暫時停權 → 永久終止**。

---

## 對應的系統工具

本流程對得上平台現有功能，不需另外開發：

- 檢舉：代購上架、評價、連線公告、賣家帳號
- 處置：軟刪除商品 / 下架代購 / 結束連線 / 賣家停權
- 管理員後台審核介面
