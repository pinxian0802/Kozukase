# 新增代購流程改版（設計）

日期：2026-06-25
狀態：設計待實作

## 背景與目標

目前賣家新增代購是兩關流程：先搜尋／新增「商品」（共用目錄），再填「代購」。痛點：

1. **麻煩**：要先選或「新增商品」（一個獨立的商品表單），才能填代購。
2. **兩個名稱搞混**：「商品名稱」與「代購標題」。
3. **兩套圖搞混**：「官方商品圖」與「代購圖片」，賣家把自己要用的圖塞進商品官方圖。

根因：「商品」這層共用目錄對賣家是隱形的，卻要他先理解並維護它。

定位前提：共用商品（同一商品聚合多賣家比價）重要但可弱化。

## 方向

- **保留「選商品」這一步**：讓會搜尋的賣家自己把代購掛到既有商品上 —— 這些情況比價與許願自然正確，不需任何後台去重。
- **把「新增商品」摺進代購表單**：找不到商品時，不再進一個獨立的商品表單，而是直接進代購表單，商品欄位內嵌其中，送出時才默默生商品。
- **完全不做**：AI 去重、後台合併、資料庫結構變更。買家端（許願、收藏、瀏覽、比價）完全不動。

## 賣家流程

**第一步：選商品（沿用現有搜尋）**

賣家搜尋自己的商品：

- **有找到 → 點選**：進代購表單；「商品資訊」區顯示該商品的卡片（唯讀）。這筆代購掛到既有 `product_id`，比價與許願正確接上。
- **沒找到 → 點「沒有我的商品」**：原「新增商品」按鈕文字改為「沒有我的商品」；點擊直接進代購表單；「商品資訊」區顯示可填欄位，且**商品名稱自動帶入賣家剛剛搜尋輸入的字**。送出時才建立新商品。

**第二步：代購表單（兩種情況共用同一張表）**

欄位由上到下：

1. 基本資訊：代購標題、圖片（只有這一套，賣家自己的）
2. 價格與庫存：價格、庫存
3. **商品資訊（中間這塊）**
   - 選了既有商品：顯示該商品卡片（唯讀），並提供「重新選擇」。
   - 新增：可填欄位 —— 商品名稱（必填，預設帶入搜尋字）、品牌、型號、分類、商品國家（皆選填）。品牌沿用現有即時新增。
4. 規格
5. 出貨與檔期：預計出貨日、截止日
6. 來源與說明：貼文連結、說明

官方商品圖整個移除；賣家只上傳一套（代購）圖。

## 背後行為

- **選既有商品**：代購直接掛該 `product_id`，不建立商品、不動其圖片。
- **新增商品**：送出時用「商品資訊」欄位建立商品（不含官方商品圖），再把代購掛上去。**把賣家上傳的第一張代購圖，於上架時再以 `product` 用途上傳一份作為商品代表圖**（寫入該商品 `product_images` 並回填 `catalog_image_id`），讓買家商品頁／搜尋／熱門沿用現有顯示機制有圖（見 `docs/notes/product-image-fallback.md`）。註：商品圖 API 僅接受 `images/product/...` 路徑的金鑰，無法直接沿用代購圖（`images/listing/...`）的金鑰，故代表圖是一份獨立副本，非共用同一檔案；也因此沒有跨表共用檔案的孤兒清理顧慮。
- 沿用現有「先建 draft listing → 上傳圖 → confirm 圖 → publish」與失敗補償回滾順序。

## 痛點解決對照

- 痛點一（麻煩）：新增商品不再是獨立一關，搜尋後直接進一張表填完；選既有商品則只是顯示卡片。
- 痛點三（兩套圖）：賣家永遠只上傳一套代購圖。新增時商品代表圖自動借第一張；選既有商品時商品本來就有圖，賣家仍只傳自己這套，沒有第二個上傳處可放錯。
- 痛點二（兩個名稱）：本期**不處理**，商品名稱與代購標題維持各自獨立欄位（選既有商品時商品名稱來自卡片、不需填）。

## 不做 / 範圍邊界

- 不對賣家做任何商品比對／建議；不自動合併。
- 不做 AI 去重、不做後台合併工具、不改資料庫結構。
- 不改編輯既有代購頁、不改買家許願選擇器、不改買家比價／篩選邏輯。
- 商品國家（region）、分類、品牌等屬性沿用現有，餵買家端瀏覽篩選。

## 已知代價（皆為「變弱」非「壞掉」，日後可再補）

- 賣家若略過搜尋直接按「沒有我的商品」，會生出重複商品 → 該商品的比價多半只有一位賣家、重複商品逐漸累積。
- 對應的許願通知在重複情況下會變弱（賣家沒掛到買家許願的那個商品時，不會通知到許願者）。
- 草稿目前仍會生商品（因 `listings.product_id` 必填），屬已知小瑕疵，本期不改 schema。
- 商品代表圖是第一張代購圖的獨立副本（多上傳一次），等於同一張圖在 R2 存兩份；屬可接受的小成本，沿用既有孤兒清理即可，無跨表共用顧慮。

> 以上代價若日後變成痛點，再單獨開「AI 去重 + 後台合併」一案處理，與本案解耦。

## 受影響的程式（概略）

- `app/(seller)/dashboard/listings/new/page.tsx`：移除獨立的「新增商品」子表單分支，改為導向內嵌商品欄位的代購表單。
- `components/product/product-picker.tsx` / `product-search.tsx`：「新增商品／建立新商品」改文案為「沒有我的商品」，行為改為帶著搜尋字進代購表單。
- `components/listing/listing-form.tsx`：新增「商品資訊」區（卡片 / 可填欄位兩態），置於價格與庫存與規格之間；新增商品時於送出建立商品並設代表圖。
- `components/product/product-form.tsx`：原獨立商品表單在此流程不再使用（其他流程如買家許願可保留）。

## 測試重點

- 選既有商品：代購正確掛該商品；商品資訊區顯示卡片、可重新選擇。
- 沒有我的商品：商品名稱帶入搜尋字；送出建立商品且代表圖正確設定；失敗回滾涵蓋新建商品與其圖。
- 全程只有一個圖片上傳處。
- 買家端許願／收藏／瀏覽／比價不受影響。
